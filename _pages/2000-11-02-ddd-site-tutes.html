[toggle title="PHP Change to make post thumbs and feature images link to post, not overlay" variation="teal"][fancy_header3 variation="copper" textColor="#000000"]Post Image Links[/fancy_header3]

You will notice on the home page that the post thumbs and feature images DO link directly to the images. I assume this is what you would prefer images to do everywhere they are attached to an article (Post).

I have rewritten the wp-content/themes/persuasion/lib/functions/image.php file to accomplish this.

This is the actual line and code changes.  I am assuming you follow this and understand what I am getting at. :)

from this,

Line 66 'link_to' => $link_to,

Line 67 'prettyphoto' => $prettyphoto,

To this,

Line 66 'link_to' => get_permalink(),

Line 67 'prettyphoto' => false,

I have also attached a screenshot.  Notice my comment in the code:

[fancy_images width="600" height="500"]

[image title="Screenshot of code change" caption="Screenshot of code change"]http://dddcommunity.org/wp-content/uploads/2012/11/Link-to-Images.jpg[/image]

[/fancy_images]

/*BJ Changed this!!! See code following comment. This was done so that fancy imeges would link to post rather then fancy image overlay*/

/*'link_to' => $link_to, 'prettyphoto' => $prettyphoto,*/

Any time I hack the core theme or anything else, I comment "BJ Changed this!!!." Then state why,  I do this because if I update the theme wholesale, it will overwrite my changes.  By commenting this way I can search for all instances of code changes using BJ Changed this!!!. I also keep my own quirky spread sheet of changes, this is backup.

So moving forward you do not need to change any code in the theme, I just wanted you to know what I did.  All thumbnails and images attached to posts and displayed in the blog shortcode, such as the library page, will link to the post (article).
